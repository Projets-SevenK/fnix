# Pixel-perfect Integration Skill — FNIX

## Required process
1. Read `prototype/README.md`.
2. Read `prototype/fnix-maquette.html` if present.
3. Identify sections, typography, colors, spacing, buttons, cards, form layout and mobile behavior.
4. Extract reusable UI patterns before coding.
5. Preserve the approved visual feeling, not necessarily every generated class from the prototype.

## Visual constraints
- Keep black dominant.
- Use white for readability.
- Use FNIX blue only for accents: buttons, focus states, badges, highlights.
- Keep the `44` visible but controlled.
- Mobile-first.
- No final product photo invention. Use clean placeholders when photos are missing.

## Integration priorities
- Hero section.
- Product/drop section.
- Brand story section.
- Values section.
- Order CTA.
- Wero explanation.
- Responsive mobile layout.

## Accessibility checklist
- Buttons and links must be keyboard focusable.
- Inputs must have labels.
- Contrast must be readable.
- Decorative `44` backgrounds should not interfere with reading.
